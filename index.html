Overview

Overview of Joker (Core Features).

(Category Selection): Allows users to choose dynamic joke categories through 4x4 keypad input.
(Cloud Translation): Fetches Korean translations via Make.com API and provides a bilingual display interface.
(Smart Joke Retrieval): Offloads heavy JSON parsing to a proxy server for efficient data handling.
(Interactive Logging): Rates jokes based on user feedback and logs data to Google Sheets for analysis.

Method & Engineering

Our project focuses on a robust IoT architecture to ensure seamless network communication and efficient data processing. Key contributions include:

1. Cloud-Offloaded Proxy Architecture

To overcome the ESP32's memory constraints, we engineered a Server-Side Proxy using Make.com. Instead of parsing complex JSON data directly on the microcontroller, the proxy pre-processes the data and delivers only essential text strings to the device.
*Architecture: ESP32 (Client) ↔ Make.com (Smart Proxy) ↔ Joke API & Google Sheets

2. Cloud-Powered Bilingual Display

We implemented a Dual-Output Localization System. The proxy integrates Google Sheets translation services, enabling the device to display English text on the LCD while simultaneously streaming Korean translations to the Serial Monitor (or secondary display) with zero local memory overhead.

3. Robust Fault Tolerance

To guarantee system availability, we developed an Infinite Retry Logic. The system autonomously detects network timeouts or API failures (e.g., 'Error' response) and re-initiates connection attempts in a non-blocking loop until a valid response is secured.

4. Dynamic Category & Feedback System

  • Dynamic API Querying: Mapped physical keypad inputs to dynamic URL parameters for real-time category filtering.
  • Cloud Logger: Automated data logging pipeline that sends user ratings directly to Google Sheets for usage analysis.
